# 项目需求文档

## 1. 项目背景

算法、计算机网络、操作系统和数据库知识点数量较多，传统笔记工具缺少面试式追问、薄弱项记录和复习任务生成能力。本项目尝试使用 LLM 和受控工具调用，构建一个用于个人面试学习的后端系统。

## 2. 目标用户

正在准备计算机开发或 AI 应用开发实习面试的学生。

## 3. 核心问题

- 知识笔记分散，难以按主题检索。
- 缺少模拟面试和递进式追问。
- 回答后的薄弱项无法自动沉淀。
- 对话过长后上下文成本上升。
- AI 输出不稳定，需要结构化验证和错误兜底。

## 4. MVP 功能

### 4.1 笔记管理

- 新建、查询、修改和删除笔记
- 按分类和掌握程度筛选

### 4.2 AI 面试

- 根据知识点生成问题
- 对回答进行结构化评价
- 生成后续追问
- 支持普通和流式响应

### 4.3 工具调用

- search_notes
- get_weak_topics
- create_review_task

### 4.4 上下文管理

- 保留最近 8–12 条消息
- 历史消息压缩为摘要
- 设置最大 Agent 调用轮数
- 长期知识通过检索按需注入

### 4.5 RAG

- 导入 Markdown 笔记
- 文本切分与相似度检索
- 回答返回来源标识

## 5. 非功能需求

- API Key 不进入 Git。
- 输入参数必须校验。
- 核心错误必须返回明确状态。
- 模型调用需要超时和有限重试。
- 工具调用需要白名单和最大轮数。
- 核心接口需要 pytest。
- README 应允许他人复现。

## 6. 两周内明确不做

- 用户注册和复杂权限系统
- 多租户和支付
- 移动端 App
- 复杂前端
- 多 Agent 协作系统
- 模型训练或微调
- 生产级高并发
- Kubernetes 和微服务

## 7. 最终验收

- 服务可以启动。
- 笔记 CRUD 可用。
- LLM 普通和流式接口可用。
- 至少三个工具可被 Agent 调用。
- 上下文窗口和摘要压缩可演示。
- RAG 可检索 Markdown 笔记并返回来源。
- 错误路径有基本测试。
- 项目可部署或有完整部署说明。
- 开发者能解释架构、失败案例和 AI 生成代码的审查过程。
